## Recommended Launch Path

### Guided Skillmap Session

Use this for most classrooms, workshops, and activations. Learners follow the MakeCode Arcade pathway in order and build one concept at a time. This path gives the clearest student on-ramp and keeps the experience cumulative across stages.

## Alternate Launch Options

### Full Project Import

Use this when you want learners, educators, or contributors to inspect the full project after the guided build.

1. Open MakeCode Arcade.
2. Choose the GitHub import option.
3. Paste the repository URL.
4. Open the project in the simulator.

This option is best after the skillmap, not before it, unless your group is already comfortable navigating a full project surface.

## Optional Facilitation Supports

- use the Garage Shakedown test track when you want a fast compare-and-explain checkpoint
- prepare a short projected reminder of the three score lenses: performance, efficiency, and strategy
- keep a one-slide or one-page version of the team roles nearby if career connections are a core goal
- keep one backup demo device ready for fast recovery in live events

## Stage-Based Remix Ideas

- **The Garage:** tune setup values or improve onboarding feedback
- **On the Road:** adjust clean-driving rewards, pit stop prompts, or weather effects
- **The Finish Line:** improve result summaries, reflection prompts, or the tone of the closing message

The best remix prompts are the ones that can be explained in a few sentences, pointed to in one small part of the project, and tested quickly in the simulator.

## Follow-Up Ideas

- invite teams to remix art, scoring, or setup choices after the core path is complete
- ask learners which role they want to learn more about and why
- share the repository link with educators who want to inspect the project structure more deeply
- use the project wiki for contributor-facing workflow notes, planning context, and implementation details
