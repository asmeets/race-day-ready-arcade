Use this section to understand the core ideas behind the experience before you teach it.

## Core Foundations

### Engineering Tradeoffs

The project is built around the idea that design choices create gains and costs at the same time. More speed can reduce control. Better efficiency can change performance. Safer choices can affect race outcomes.

### Iteration Through Testing

Learners should change something, test it, notice the result, and revise. That cycle matters more than reaching one perfect setup.

### Career Connection Through Action

Roles such as software engineer, test engineer, strategist, and sustainability lead become easier to understand when learners use that kind of thinking in the game.

## Design Principles

- keep the experience readable and projector-friendly
- keep core values and code patterns visible enough to remix
- keep failure states recoverable so iteration feels normal
- keep the learning value tied to observable game outcomes

## What Educators Should Listen For

- learners naming a change and its effect
- teams disagreeing productively about the best tradeoff
- comparisons that use evidence instead of guesses alone
